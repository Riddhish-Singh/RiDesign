Exported HTML from RiDesign is a single-file static site. This lesson shows practical ways to share it fast: quick blob share, GitHub Pages, Netlify, Vercel, or classic FTP/hosting panel. Pick the workflow that fits your level.

Quick recap ‚Äî how to export
Download .html
Saves a single file with embedded images/videos. Best for manual uploads or local previews.
Copy HTML
Copies full page HTML to clipboard. Paste into an editor (VS Code) or CMS.
Share Site
Creates a temporary blob URL for instant share/testing in a new tab (ideal for quick feedback).
Option A ‚Äî Temporary quick share (fastest)
  1. In builder click Share Site. A blob link appears. Share that link with anyone for quick preview.
  2. Note: the blob URL is temporary and only available in the current browser session. Use it for demos, not long-term hosting.
Pro tip: If you need longer-term temporary hosting, use Netlify Drop (below) or GitHub Pages.
Option B ‚Äî GitHub Pages (static, free)

Good when you want a stable public URL and you are comfortable with Git or GitHub UI.

  1. Open GitHub ‚Üí New repository ‚Üí choose a name (e.g., my-site), make it public.
  2. Download your site from RiDesign as index.html.
  3. Push to the repo (CLI):
// in terminal (inside folder with index.html)
git init
git add index.html
git commit -m "initial RiDesign export"
git branch -M main
git remote add origin https://github.com/USERNAME/REPO.git
git push -u origin main
  1. Go to repository ‚Ä∫ Settings ‚Ä∫ Pages (or "Code and automation" ‚Üí Pages). Select branch main / folder / (root), then Save. Your site will be available at https://USERNAME.github.io/REPO/ a few minutes later.
Notes: Use index.html as filename. For a custom domain, add a CNAME file or configure in Pages settings.
Option C ‚Äî Netlify Drop (drag & drop) ‚Äî easiest for non-devs

Netlify lets you drag the exported index.html into their Drop UI and instantly publishes a site with a free URL.

  1. Go to Netlify ‚Üí Sites ‚Üí "Add new site" ‚Üí "Deploy with drag and drop".
  2. Drop your index.html file or a zip of the exported site. Netlify builds and gives you a live URL.
Optional: connect a GitHub repo for continuous deploys when you push changes.
Option D ‚Äî Vercel (import from Git or drag & drop)

Vercel also supports static sites. Import your repository or drag & drop a folder. Best for quick previews and production-ready hosting.

  1. Create a Vercel account and import the Git repository that contains your index.html, or use their drag & drop.
  2. Vercel sets up a project and gives an instant URL; connect a custom domain if needed.
Option E ‚Äî Traditional hosting (FTP / cPanel)

If you have hosting control panel access, upload index.html to the public_html or www folder using the hosting file manager or FTP/SFTP.

// example (using sftp)
sftp user@your-host.com
put index.html public_html/index.html
exit
Then visit your domain. Use SFTP for secure uploads.

Troubleshooting
  • Blob link doesn't open for others: use Netlify / GitHub Pages instead.
  • Embedded videos not playing on some hosts: ensure the exported file contains data URLs (RiDesign does this when you upload files in the builder).
  • Page looks different on host: clear cache or inspect the index.html you uploaded for missing assets.
